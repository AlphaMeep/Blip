# ShoulderBot Eye ("Blip")

A Portal-style expressive robot eye on a GC9A01 1.28" round display. It has moods,
a daily disposition, sound, idle micro-behaviours, bot-to-bot BLE socialising, a
persistent "pet" memory, and — on the Waveshare build — full 6-axis motion sense.

---

## Builds

There are three firmware variants plus two browser tools. Pick the `.ino` that
matches your hardware.

| Sketch | Board | Highlights |
|---|---|---|
| `waveshare/shoulder_bot_eye_waveshare/…ino` | **Waveshare ESP32-S3-Touch-LCD-1.28** | **Flagship.** Onboard QMI8658 IMU (gestures + always-upright), LiPo battery manager, 10+2 themes. No external wiring needed except a speaker. |
| `s3/shoulder_bot_eye_s3/…ino` | ESP32-S3 + GC9A01 (generic) | NUS BLE link to the web dashboard. No ANCS. **Stable archive.** |
| `shoulder_bot_eye.ino` | Original ESP32 + GC9A01 | iOS **ANCS** notification mirroring. The first build. |

### Browser companions (no install, open in Chrome/Edge)
- **`blip_network.html`** — Web Bluetooth dashboard: connect over BLE *or* USB serial,
  live serial monitor, per-bot customisation (name/theme/personality/voice…), a
  friendship map of who's bonded with whom, bond sparklines, and battery readouts.
- **`blip_sim.html`** — fully offline simulator of the eye (canvas + Web Audio) with
  every animation and sound, plus an **animation creator** that exports Arduino code.

> `preview.html` was removed (it modelled an older eye). Serve the HTML over
> `http://localhost` (e.g. `python -m http.server 8080`) if you want Web Bluetooth;
> the **USB** button works straight from `file://`.

## Waveshare build — hardware

Almost everything is onboard. You only add a speaker.

| Function | GPIO | Notes |
|---|---|---|
| LCD GC9A01 | MOSI 11, SCLK 10, CS 9, DC 8, RST 12 | set in TFT_eSPI (see below) |
| LCD backlight | 40 | digital on/off (not PWM — see speaker note) |
| QMI8658 IMU | SDA 6, SCL 7, INT1 47, INT2 48 | I2C `0x6B`; accel + gyro |
| Battery sense | 1 | onboard divider; calibratable |
| **Speaker** | **17** | straight off the GPIO via `tone()` — **no amp.** Wire speaker ↔ GND (a passive piezo or small dynamic speaker). Optional ~100 Ω in series. |
| BOOT button | 0 | sleep/wake |
| USB | Type-C (CH343P) | Serial works over USB; no TX/RX pins needed |

> **Backlight is plain on/off, not PWM.** `tone()` on ESP32 Arduino core 3.x is
> generated by the LEDC peripheral; driving the backlight with LEDC too starves it
> and the speaker goes silent. So the backlight is digital (on awake, off asleep).

### TFT_eSPI setup (required, or the screen stays black)
The repo ships `Setup_ShoulderBot_Waveshare.h`. Put it in the TFT_eSPI library and
select it in `User_Setup_Select.h`:

```cpp
//#include <User_Setup.h>
#include <User_Setups/Setup_ShoulderBot_Waveshare.h>
```

It sets the GC9A01 driver, the pins above, and crucially **`USE_HSPI_PORT`** — omit
that and `tft.init()` crashes (null SPI pointer → bootloop). Do **not** define
`TFT_BL`; the sketch owns GPIO40.

### Power
The board can brown-out on a weak USB port/cable when the display + BLE radio spike
at boot (symptom: ~1 Hz backlight blink, never enumerates serial). **A LiPo on the
battery header fixes it** (it buffers the current). Otherwise use a 5 V wall adapter
and a known-good *data* cable. BLE TX power is already turned down in firmware to help.

## Configuration

Edit the `#define`s near the top of the sketch and re-flash. (Most are also
changeable live over serial and saved to flash — see Serial Commands.)

### `EYE_THEME` — iris colour
Static palettes `0`–`9`, plus two dynamic modes set at runtime via `set theme`.

| | | | |
|---|---|---|---|
| `0` Cyan | `1` Green | `2` Purple | `3` Orange |
| `4` Red | `5` Ice | `6` Pink | `7` Gold |
| `8` Teal | `9` White | `10` Rainbow cycle | `11` Mood-reactive |

### `PERSONALITY` — emotional bias
`0` Balanced · `1` Cheerful · `2` Grumpy · `3` Anxious · `4` Stoic

### `EXTROVERSION` — how eagerly it fills silence
`0` Introverted · `1` Balanced · `2` Extroverted (scales all idle/major gaps)

### `TALKATIVENESS` — richness of sound
`0` Quiet · `1` Balanced · `2` Talkative

### Identity & voice
`BOT_NAME`, `VOICE_PITCH` (0 deep / 1 normal / 2 high), `ACCENT_COLOR`,
`SOUND_STYLE` (0 melodic / 1 chirpy / 2 droning / 3 soft / 4 distorted), `QUIRK`.

### Waveshare-specific
| Define | Purpose |
|---|---|
| `USE_IMU` | `1` = QMI8658 gestures + upright mode |
| `UPRIGHT_DEFAULT` | start with gravity-leveled visor on/off |
| `BATT_MONITOR` | `1` = battery %, tired-mode, critical auto-sleep |
| `BATT_DIVIDER_DEFAULT` | divider ratio (calibrate live with `set battcal`) |
| `SLEEP_TIMEOUT_MIN` | inactivity sleep (0 = never) |

## Motion & gestures (Waveshare)

The QMI8658 replaces the old microphone. Pick the bot up, tap it, shake it:

| Gesture | Reaction |
|---|---|
| **Single tap** | `doHeard` — "huh?" startled look |
| **Double tap** | `doGreet` — friendly hello (+mood) |
| **Triple tap** | `doSecret` — Easter egg |
| **Shake** | `doShocked` — **pupil dilation + recoil scale with how hard you swing** |
| **Spin** (gyro) | `doDizzy` — woozy spiral |
| **Flip upside-down** | `doFlipReact` — "…am I upside down?" |
| **Free-fall / drop** | `doWhee` — stomach-drop, pupils blown wide |
| **Pick-up** | `doCurious`; **put-down** → settle blink |
| **Tilt (on a desk)** | glances "downhill" |

**Weighted-eye inertia** — the eye sloshes a few pixels with real weight when the
bot is moved, then self-centres. Invisible while still.

**Always-upright mode** — `upright on` makes the visor stay level with gravity no
matter how the board is rotated. Run `upright cal` once it's mounted to zero the
level reference for your orientation (`set uprightsign -1` flips it if mirrored).

## Physical controls

- **BOOT button (GPIO 0):** press to toggle sleep/wake.
- **Motion:** all the gestures above (Waveshare). The original/S3 builds use a
  tactile switch and (optionally) a microphone or accelerometer instead.

## Bluetooth

### Web dashboard (all builds)
The bot runs a Nordic UART (NUS) service. Open `blip_network.html`, hit **Bluetooth**
(served over localhost) or **USB** (works from `file://`), and you get live status,
a serial console, customisation, and the friendship map.

### iOS notifications (original build only)
The `shoulder_bot_eye.ino` build subscribes to **ANCS** — pair it in iOS Bluetooth
settings and every phone notification makes the eye snap to alert and show the title.
(If your ESP32 core doesn't expose the GAP security headers, ANCS auto-disables and
everything else still works.)

### Bot-to-bot ("Tamagotchi mode")
Two or more Blips detect each other by BLE scan and react based on a **bond** that
grows over repeated meetings. Tiers: Unknown → Familiar → Friend → Trusted → Bonded
→ Kindred. They play rock-paper-scissors and tic-tac-toe, mirror each other's
expressions, can inherit traits, and sing happy birthday on a peer's "birthday"
(the date it was first powered on, stored write-once in flash).

## Battery manager (Waveshare)

| Command / behaviour | Detail |
|---|---|
| `batt` | prints %, voltage, raw ADC, divider |
| `set battcal <ratio>` | calibrate: measure VBAT with a meter, set the ratio |
| Tired mode | below 20% the bot slows down |
| Critical | below ~8% it warns and sleeps to protect the cell |
| Implausible reading | (no battery / USB only) is treated as 100% — no false sleep |

## Serial commands

Connect at **115200 baud**, case-insensitive, type `help` for the live list.

- **Expressions:** `happy mad sad shock wink sus dead love smug proud surprised secret boot rec dis laugh cool musing`
- **Personality:** `nervous curious think sleepy alert nod scan panic glance dglance`
- **Micro-behaviours:** `heard yawn squint distract selfcon eyeroll dblink eyerub peek`
- **Chains:** `confused flustered reluctant surphappy madsad relief`
- **Music:** `hum` (13 tunes incl. Zelda, Portal, Pokémon, Mega Man, Undertale), `beatbox`
- **Minigame:** `celebrate lose draw rps ttt chat`
- **Social:** `greet excited acting`
- **Utils:** `mute bored sleep wake mood status bonds bonds reset help`
- **Waveshare HW:** `upright [on/off/cal]  level  imu  temp  batt`
- **Live config (`set <key> <value>`):** `voice accent posture sound mood theme(0-11) pers ext talk bond name stage battcal uprightsign bl`

## Automatic behaviour

Two schedulers run continuously:

- **Idle micro-behaviours** — small ticks (blink, glance, yawn, squint, zone-out,
  stretch, processing, self-talk…). On the Waveshare build these are heavily spaced
  out: most idle ticks do nothing, so the bot stays calm and still for long stretches.
- **Major emotions** — the full expression pool, gated by personality, day-mood, a
  mood-momentum system (recent good/bad events bias what comes next), and an action
  cooldown so behaviours never stampede. The `MAJOR_GAP_MULT` constant stretches the
  spacing further.

**Boredom** shortens the gaps the longer it's ignored; after ~15 min it throws
attention-grabbing tantrums. **Mood scar** — a run of negatives visibly dims the eye
for a few minutes; celebrations heal it. **Tired mode** (low battery) slows it down.

---

## Pet memory (NVS)

Stored in flash, survives power cycles: boot count, total interactions, learned
traits, per-peer bond/RPS history, birthday, theme, and the Waveshare calibrations
(battery divider, upright zero). Day mood is re-rolled each boot for variety.

## Notes

- **No WiFi** — BLE only, by design.
- **Sprite** is a full 240×240 8-bit buffer (in PSRAM on the S3 boards); falls back
  to direct TFT draw if allocation fails.
- **Breathing** is deliberately *very* subtle (~10 breaths/min, sub-pixel) so it
  reads as alive, not creepy.
- **Animations are pure eye dynamics** — pupil dilation, position, and visor motion —
  rather than drawn-on faces, so the iris visibly stretches and reacts (matching the
  simulator). A few overlays (heart, sunglasses, RPS move) are intentional exceptions.
